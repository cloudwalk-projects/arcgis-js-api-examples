Настройки среды анализа Portal

Настройки среды анализа в Portal for ArcGIS и ArcGIS Online.

Перезаписать слой результата

Возможность перезаписи существующих слоев. Если опция отмечена, то в случае наличия слоя с таким же именем в ваших ресурсах, он будет перезаписан.

  • Отмечена - В случае наличия слоя с таким же именем в ваших ресурсах, он будет перезаписан. Выходные данные будут находиться в системе координат входного набора данных.
  • Не отмечена - В случае наличия слоя с таким же именем в ваших ресурсах появится сообщение об ошибке, и для выполнения инструмента вам потребуется переименовать итоговый слой. Используется по умолчанию.

Закрыть виджет анализа после запуска анализа.

Возможность оставить открытым и активным диалоговое окно анализа во время выполнения инструмента. Эта настройка позволит вам легко вносить изменения в свои параметры анализа, в то время как предыдущее задание продолжит выполняться. Анализ - это итеративный процесс, и, оставляя открытым диалоговое окно инструмента, вы сможете быстро изменить параметры и перезапустить анализ.

  • Отмечена - Окно инструмента анализа закроется, а фокус снова перейдет на таблицу содержания. Используется по умолчанию
  • Не отмечена - Окно инструмента анализа останется открытым и активным.

Хранить результаты анализа в виде сервисов объектов

Возможность указать, будут ли результаты вашего анализа создаваться в виде векторного слоя либо в виде набора объектов.

  • Отмечена - Результаты вашего анализа будут создаваться в виде векторного слоя, а в содержании появится новый элемент. Используется по умолчанию.
  • Не отмечена - По результатам анализа будет создан набор объектов. Наборы объектов сохраняются в вашей веб-карте.

Выходная система координат

Настройка среды растрового анализа, использующаяся для задания системы координат выходного растра.

  • Как у входных данных - У результатов вашего анализа будет такая же система координат, как у входных данных. Используется по умолчанию.
  • Как указано - Выберите систему координат для результатов вашего анализа. Если выбрана эта опция, щелкните значок глобуса, чтобы выбрать из списка известных систем координат нужную, либо добавьте в соответствующем окне WKID системы координат.
  • Название <слоя> - Выберите существующий слой вашей веб-карты, который задаст систему координат результатов вашего анализа.

Система координат обработки

Настройка среды для GeoAnalytics, позволяющая задать систему координат для вашего анализа, будет применяться в течение его выполнения.

Для GeoAnalytics некоторые виды анализа требуют, чтобы обработка выполнялась в системе координат проекции. Для любого инструмента, в котором реализован бининг данных или линейные единицы, необходима система координат проекции.

  • Как у входных данных -Ваш анализ будет выполнен в системе координат входных данных. Используется по умолчанию.
  • Как указано - Выберите систему координат для выполнения вашего анализа. Если выбрана эта опция, щелкните значок глобуса, чтобы выбрать из списка известных систем координат проекции нужную либо добавьте в соответствующем окне WKID системы координат.
  • Название <слоя> - Выберите существующий слой вашей веб-карты. Система координат выбранного слоя будет использоваться в качестве системы координат при выполнении вашего анализа.

Экстент

Настройка среды, использующаяся для задания экстента или границы, которая будет применена при выполнении анализа. Все целиком находящиеся в ее пределах или пересекающие заданный экстент объекты будут включены в анализ.

  • По умолчанию - Используется заданный инструментом экстент.
  • Как указано - Введите вручную координаты для задания экстента.
  • Название <слоя> - Выберите существующий слой вашей веб-карты. Пространственный экстент выбранного слоя будет использоваться в качестве экстента вашего анализа.

Растр привязки

При выполнении анализа растра задание среды Растр привязки уточнит экстент выходного растрового слоя так, чтобы ячейки были выровнены так же, как у заданного слоя Растра привязки.

Размер ячейки

При выполнении анализа растра с помощью этой настройки среды задается размер ячейки или разрешение, использующееся при создании выходного растрового слоя. Выходное разрешение по умолчанию определяется, исходя из наибольшего размера ячейки из входных растровых слоев.

  • Минимальный из всех входных – Наименьший размер ячейки из всех входных слоев.
  • Максимальный из всех выходных – Наибольший размер ячейки из всех входных слоев. Используется по умолчанию.
  • Как указано - Задайте численное значение размера ячейки. Если выбрана эта опция, значение по умолчанию будет равно 1.
  • Название <слоя> - Устанавливается размер ячейки выбранного растрового слоя.

Маска

При выполнении анализа растра используйте параметр среды Маска, чтобы задать слой, который будет определять область вашего интереса. Только ячейки, попадающие внутрь маски анализа, будут участвовать в анализе.

  • Маской может быть растровый или векторный слой.
  • Если маской анализа является растр, все содержащие значения ячейки будут считаться определяющими маску. Ячейки в растре маски со значениями NoData будут рассматриваться как находящиеся вне маски, и в слое результатов анализа соответствующим ячейкам будет присвоено значение NoData.
  • Если маской анализа является векторный слой, он будет внутренне конвертирован в растр при выполнении. По этой причине вам следует убедиться, что Размер ячейки и Растр привязки установлены соответствующим образом для анализа.
