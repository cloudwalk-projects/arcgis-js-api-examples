Calcular Densidade

The Calculate Density tool creates a density map from point features by spreading known quantities of some phenomenon (represented as attributes of the points) across the map. The result is a layer of areas classified from least dense to most dense.

For point input, each point should represent the location of some event or incident, and the result layer represents a count of the incident per unit area. A higher density value in a new location means that there are more points near that location. In many cases, the result layer can be interpreted as a risk surface for future events. For example, if the input points represent locations of lightning strikes, the result layer can be interpreted as a risk surface for future lightning strikes.

Other use cases of this tool include the following:

  • Creating crime-density maps to help police departments properly allocate resources to high crime areas.
  • Calculating densities of hospitals within a county. The result layer will show areas with high and low accessibility to hospitals, and this information can be used to decide where new hospitals should be built.
  • Identifying areas that are at high risk of forest fires based on historical locations of forest fires.
  • Locating communities that are far from major highways in order to plan where new roads should be constructed.

Escolha uma camada de ponto da qual calcular densidade

The point layer on which the density will be calculated.

Além de escolher uma camada de seu mapa, você pode escolher Procurar Camadas na parte inferior da lista suspensa para procurar por seu conteúdo para um conjunto de dados de compartilhamento de arquivo big data ou camada de feição.

Escolha um ou mais campos no qual calcular densidade

Provide one or more fields specifying the number of incidents at each location. For example, if you have points that represent cities, you can use a field representing the population of the city as the count field, and the resulting population density layer will calculate larger population densities near cities with larger populations.

The density for the count of points will always be calculated. If no fields are selected, each location will be assumed to represent a single count.

Escolha uma forma de bin

The shape of bins that points are aggregated into and used for the density calculation.

Selecione o tamanho de bin para cálculos de densidade

The size of the bins generated. For a square bin, the size represents the height of the square. For hexagon, the size represents the height of the hexagon (from parallel sides).

Selecione o peso de densidade para aplicar

The type of weighting applied to the density calculation. There are two options:

  • Uniform—This calculates a magnitude per area.
  • Kernel—Applies a kernel function to fit a smooth tapered surface to each point.

Selecione o tamanho da vizinhança

The size of the area used to calculate the density. The neighborhood size must be larger than the bin size.

Escolha as unidades de saída para densidade

The desired output units of the density values. By default, this is set to the units in your map (for example, square meters). If density values are very small, you can increase the size of the area units (for example, square meters to square kilometers) to return larger values. The values only scale the result.

Calcular densidade utilizando partes de tempo (opcional)

Se o tempo estiver habilitado na camada de ponto de entrada e for do tipo momento, você pode analisar utilizando partes do tempo. Há três parâmetros que você pode configurar ao utilizar o tempo:

  • Intervalo de Tempo
  • Intervalo de Tempo
  • Tempo de Referência

Imagine que você tinha dados que representaram um ano em tempo e você desejou analisar partes semanais. Para fazer isto você iria configurar Intervalo de Tempo para 1 semana.

Imagine que você tinha dados que representaram um ano em tempo e você desejou analisar utilizando a primeira semana do mês. Para fazer isto você iria configurar Intervalo de Tempo para 1 semana, Etapa de Tempo para 1 mês e Tempo de Referência para 1 de janeiro, as 12:00 am.

Intervalo de tempo no qual calcular densidade

O intervalo de tempo utilizado para fatiar os dados. O intervalo de tempo pode ser utilizado sozinho ou utilizado com o parâmetro Etapa de Tempo ou Tempo de Referência.

Se você desejou criar fatias de tempo que ocorreram toda segunda-feira as 9:00 a.m. até 10:00 a.m., Intervalo de Tempo seria 1 hora, Etapa de Tempo seria 1 semana e Tempo de Referência seria 9:00:00 a.m. em uma segunda-feira.

Espaço de tempo no qual calcular densidade

A etapa utilizada para calcular uma fatia de tempo. Etapa de Tempo pode ser utilizado sozinha ou com Intervalo de Tempo, com Tempo de Referência ou com ambos Intervalo de Tempo e Tempo de Referência.

Se você desejou criar fatias de tempo que ocorreram toda segunda-feira as 9:00 a.m. até 10:00 a.m., Intervalo de Tempo seria 1 hora, Etapa de Tempo seria 1 semana e Tempo de Referência seria 9:00:00 a.m. em uma segunda-feira.

Tempo de referência utilizado para alinhar partes de tempo

O data e o tempo utilizado para alinhar as fatias do tempo. Etapa de Tempo irá iniciar e continuar para trás do tempo referenciado.

Se você desejou criar fatias de tempo que ocorreram toda segunda-feira as 9:00 a.m. até 10:00 a.m., Intervalo de Tempo seria 1 hora, Etapa de Tempo seria 1 semana e Tempo de Referência seria 9:00:00 a.m. em uma segunda-feira.

SpatialReference (wkid)

Este é um parâmetro temporário do pré lançamento para configurar a referência de processamento espacial. Muitas ferramentas de big data exigem que um sistema de coordenada plana seja utilizada como a referência espacial para processamento. Por padrão, a ferramenta utilizará o sistema de coordenadas de entrada mas falhará se for um sistema de coordenadas geográficas. Para configurar um sistema de coordenadas plana, insira o WKID. Por exemplo, Web Mercator seria inserido como 3857.

Escolher armazenamento de dados

Este é um parâmetro temporário para pré lançamento para configurar o armazenamento de dados onde os resultados são salvos. Ambos os armazenamentos de dados são parte do ArcGIS Data Store. Se você selecionar o armazenamento de dados spatiotemporal mas não ter um instalado, a ferramenta falhará.

Nome da camada resultante

Este é o nome da camada que será criada em Meu Conteúdo e adicionada ao mapa. O nome padrão é baseado no nome da ferramenta e o nome da camada de entrada. Se a camada já existe, a ferramenta falhará.

Ao utilizar a caixa suspensa Salvar resultado em, é possível especificar o nome de uma pasta em Meu Conteúdo onde o resultado será salvo.
