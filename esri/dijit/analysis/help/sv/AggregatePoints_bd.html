Aggregera punkter

Det här verktyget arbetar med ett lager med punktgeoobjekt och ett lager med områdesgeoobjekt. Områdesgeoobjekten som används som indata kan vara från ett polygonlager eller så kan de vara kvadratiska eller sexkantiga lagringsplatser som beräknas när verktyget körs. Först avgörs vilka punkter som hamnar inom respektive angivet område. Efter att den geografiska analysen för punkter inom olika områden är färdig, beräknas statistik för alla punkter i respektive område. Den mest grundläggande statistiken är räkningen av antalet punkter i området, men du kan även skaffa annan statistik.

Anta till exempel att du har punktgeoobjekt i form av platser för kaféer och områdesgeoobjekt i form av län, och nu vill summera kaffeförsäljningen per län. Om vi antar att kaféerna har attributet TOTAL_SALES kan du beräkna summan av alla TOTAL_SALES i varje län, minsta och högsta TOTAL_SALES i varje län eller annan statistik som antal, omfång, standardavvikelse och varians.

Verktyget kan också arbeta med data som är tidsaktiverade. Om tid är aktiverat på indatapunkterna är alternativen för tidsdelning tillgängliga. Med tidsdelning kan du beräkna förhållandet för punkter inom olika områden medan du tittar på en viss tidsdel. Du kan t.ex. titta på timintervaller, vilket ger utdata för varje timme.

Med tid kan du till exempel ha punktgeoobjekt för varje utförd transaktion vid olika platser för caféer och inget områdeslager. Data har registrerats under ett år och varje transaktion har en plats och en tidsstämpel. Om vi antar att varje transaktion har attributet TOTAL_SALES, kan du få summan av alla TOTAL_SALES inom det intressanta rummet och tiden. Om transaktionerna är för en enda ort kan vi generera områden som är rutnät med en kilometers upplösning och se på veckotidsdelar för att summera transaktionerna i tid och rum.

Välj ett lager som innehåller punkter som ska aggregeras till områden

Punktlagret som ska aggregeras till områden.

Förutom att välja ett lager från kartan kan du välja Bläddra bland lager längst ned i listrutan för att bläddra till ditt innehåll efter ett big data-fildelningsdataset eller geoobjektslager.

Välj ett områdeslager att aggregera till

De områden som punktlagret ska aggregeras till. Du kan välja mellan ett områdeslager som du anger eller lagringsplatser som genereras när verktyget körs. När lagringsplatser skapas avgör talet och enheterna som angivits för Kvadrat kvadratens höjd och längd. För Hexagon avgör talet och enheterna som angivits avståndet mellan parallella sidor.

Välj områdeslagret att aggregera till

Områdeslagret som punkterna ska aggregeras till. Alla punkter som hamnar inom gränserna för områden i lagret räknas, och du kan välja att summera dem med valfri statistik.

Du måste ange antingen Lagringsplatsstorlek för lagringsplatser eller ett Områdeslager.

Välj avstånd för att generera lagerplatser och aggregera till

Det avstånd som används för att generera lagringsplatser att aggregera dina indatapunkter till. För en kvadratisk lagringsplats motsvarar storleken kvadratens höjd. För en hexagon motsvarar storleken hexagonens höjd (från parallella sidor).

Du måste ange antingen Lagringsplatsstorlek för lagringsplatser eller ett Områdeslager.

Aggregera med tidsdelar (valfritt)

Om tid är aktiverat på indatapunktlagret och det är av typen tidspunkt, kan du analysera med tidsdelning. Du kan ställa in tre parametrar när du använder tid:

  • Tidsintervall
  • Tidssteg
  • Referenstid

Tänk dig att du har data som representerar ett år i tiden och du vill analysera med tidsdelar på veckonivå. Det gör du genom att ange Tidsintervall som 1 vecka.

Tänk dig att du har data som representerar ett år i tiden och du vill analysera utifrån första veckan i månaden. Det gör du genom att ange Tidsintervall som 1 vecka, Tidssteg som 1 månad och Referenstid som 1 januari kl 0:00.

Tidsintervall som ska aggregeras till

Tidsintervall som används för att dela data. Tidsintervall kan användas fristående eller användas med parametern Tidssteg eller Referenstid.

Om du vill skapa tidsdelar som äger rum varje måndag från 9.00 till 10.00 skulle Tidsintervall vara 1 timme, Tidssteg skulle vara 1 vecka, och Referenstid skulle vara 9:00:00 på en måndag.

Tidssteg som ska aggregeras till

Det steg som används för att beräkna en tidsdel. Tidssteg kan användas fristående eller med Tidsintervall, med Referenstid, eller med både Tidsintervall och Referenstid.

Om du vill skapa tidsdelar som äger rum varje måndag från 9.00 till 10.00 skulle Tidsintervall vara 1 timme, Tidssteg skulle vara 1 vecka, och Referenstid skulle vara 9:00:00 på en måndag.

Referenstid som används till att justera tidsdelar

Datum och tid som används för att justera tidsdelar. Tidssteg startar från och fortsätter bakåt för referenstiden.

Om du vill skapa tidsdelar som äger rum varje måndag från 9.00 till 10.00 skulle Tidsintervall vara 1 timme, Tidssteg skulle vara 1 vecka, och Referenstid skulle vara 9:00:00 på en måndag.

Lägg till statistik (valfritt)

Du kan beräkna statistik på summerade geoobjekt. På numeriska fält kan du beräkna följande:

  • Antal
  • Medelvärde
  • min
  • max
  • Intervall
  • Varians
  • Standardavvikelse

På fält med strängar kan du beräkna följande:

  • Antal
  • Alla – Det statistiska värdet är ett slumpmässigt exempel på ett strängvärde i det angivna fältet.
All statistik beräknas på värden som inte är null. Det resulterande lagret innehåller ett nytt fält för varje statistiskt värde som beräknas. Du kan lägga till valfri mängd statistiska värden genom att välja ett attribut och statistik.

SpatialReference (wkid)

Det här är en tillfällig parameter för förhandsversionen för att ställa in den geografiska referensen för bearbetning. Många big data-verktyg kräver att ett projicerat koordinatsystem används som geografisk referens för bearbetning. Som standard använder verktyget indatakoordinatsystemet men misslyckas om det är ett geografiskt koordinatsystem. Ange WKID för att ställa in ett projicerat koordinatsystem. Till exempel anges Web Mercator som 3857.

Välj datalager

Det här är en tillfällig parameter för förhandsversionen för att ange det datalager där resultaten sparas. Båda datalagren ingår i ArcGIS Data Store. Om du väljer det rumstemporala datalagret men inte har ett installerat misslyckas verktyget.

Resultatlagrets namn

Det här är namnet på lagret som skapas i Mitt innehåll och läggs till i kartan. Standardnamnet baseras på verktygets namn och indatalagrets namn. Om lagret redan finns misslyckas verktyget.

Om du använder listrutan Spara resultat i kan du ange namnet på en mapp i Mitt innehåll där resultatet ska sparas.
